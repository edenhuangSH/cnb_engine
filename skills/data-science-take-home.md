## When to Use
When creating screening challenges for senior/PhD-level data science candidates.

## Key Patterns
- Paired notebooks: problem.ipynb (challenge) + solution.ipynb (full answer)
- pip install cell at top of each notebook
- 5-part structure: EDA → Feature Engineering → Modeling → Correction/Application → Visualization
- Clear rubric/evaluation criteria per section
- Solution shows full execution with outputs and commentary
- Use realistic industry context (company-specific data, real business problems)

## Checklist
- [ ] Include pip install cells at top
- [ ] Clear section headers with markdown cells
- [ ] Rubric with point allocations
- [ ] Solution notebook fully executed with all outputs
- [ ] Appropriate difficulty for target level (PhD = advanced statistics, novel approaches)
